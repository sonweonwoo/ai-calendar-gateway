# 게이트웨이 클라우드 배포 (Render)

앱에서 AI 키를 빼내고 서버에만 두기 위한 배포입니다.
**제가 대신 할 수 없는 부분(계정 로그인·키 입력)은 사장님이 직접 하셔야 합니다.**

---

## 왜 하는가
지금 테스터용 APK에 키를 넣으면 **APK를 뜯어 키를 꺼낼 수 있습니다.**
배포 후에는 앱에 키가 없고, 서버가 사용량·비용을 통제합니다.

---

## 1단계 — GitHub에 올리기

Render는 GitHub 저장소를 보고 배포합니다.

1. https://github.com/new 에서 새 저장소 만들기
   - 이름 예: `ai-calendar-gateway`
   - **Private(비공개)** 로 만드세요
2. 만들어진 화면에 나오는 명령을 복사해, **`C:\smart_organizer\gateway` 폴더에서** 실행
   ```bash
   cd C:\smart_organizer\gateway
   git init
   git add .
   git commit -m "AI gateway"
   git branch -M main
   git remote add origin https://github.com/<본인계정>/ai-calendar-gateway.git
   git push -u origin main
   ```
   > `.gitignore`가 있어 `node_modules`와 `.env`는 올라가지 않습니다. **키는 저장소에 남지 않습니다.**

---

## 2단계 — Render에서 배포

1. https://render.com 로그인 (GitHub 계정으로 로그인 가능)
2. **[New +] → [Web Service]**
3. 방금 만든 저장소 선택 → **[Connect]**
4. 설정은 `render.yaml`을 자동으로 읽습니다. 아래만 확인:
   - Runtime: **Node**
   - Build Command: `npm install`
   - Start Command: `npm start`
   - Plan: **Free**
5. **[Advanced] → Environment Variables** 에서 키 입력 ⭐
   - Key: **`ANTHROPIC_API_KEY`** ← **이 이름이어야 합니다**
   - Value: `sk-ant-api03-...` (콘솔에서 새로 만든 키)
   > ⚠️ 이 값은 **여기에만** 넣습니다. 저장소·앱에는 절대 넣지 않습니다.
   > ⚠️ **이름을 틀리면 AI가 조용히 멈춥니다.** `AI_API_KEY` 같은 다른 이름에 넣으면
   > 서버는 그 값을 읽지 않는데 `/health`는 여전히 정상으로 보여 원인 파악이 어렵습니다.
   > (실제로 이 사고가 있었음)
6. **[Create Web Service]** → 몇 분 기다리면 배포 완료

---

## 3단계 — 주소 확인

배포가 끝나면 위쪽에 주소가 나옵니다:
```
https://ai-calendar-gateway-xxxx.onrender.com
```

**잘 됐는지 확인**: 브라우저에서 주소 뒤에 `/health` 를 붙여 열어보세요.
```
https://ai-calendar-gateway-xxxx.onrender.com/health
```
아래처럼 나오면 성공입니다:
```json
{"ok":true,"configured":true,"models":{...},"dailyLimit":20}
```
- `"configured": true` ← **키가 잘 들어갔다는 뜻**
- `false`면 5단계 환경변수를 다시 확인하세요

---

## 4단계 — 그 주소를 알려주세요

주소를 알려주시면 제가 앱을 그 주소로 연결해 다시 빌드하겠습니다.
그러면 **앱에 키가 없는 상태로 테스터가 AI를 쓸 수 있습니다.**

---

## 알아두실 점

- **무료 플랜은 15분간 요청이 없으면 서버가 잠듭니다.**
  다음 요청 때 깨어나는 데 **30초~1분**이 걸립니다.
  (앱에서 2분까지 기다리도록 해두었지만, 테스터가 "느리다"고 느낄 수 있습니다)
  → 매달 $7 유료 플랜으로 바꾸면 잠들지 않습니다.
- 사용 한도는 **기기당 하루 20회**입니다. 바꾸려면 Render 환경변수 `DAILY_LIMIT` 수정.
- 비용 폭주 방지로 **서버 전체 하루 5000회** 상한이 걸려 있습니다(`GLOBAL_DAILY_LIMIT`).
